# Запит на міграцію проєкту на Supabase

Створіть нову версію застосунку з цього репозиторію, замінивши Firebase на Supabase. Усі наявні можливості мають зберегтися без зміни поведінки інтерфейсу чи API.

## Ключові вимоги
- **Аутентифікація:** замінити Firebase Auth на Supabase Auth (email/пароль та соціальні провайдери, якщо вони були підключені). Переконатися, що захист маршрутів Next.js працює через нові токени сесій Supabase.
- **База даних та зберігання:** перенести всі колекції/документи Firestore у таблиці Postgres Supabase. Оновити запити читання/запису на клієнті та сервері. Для файлів (якщо є) використати Supabase Storage.
- **Конфігурація клієнта:** додати SDK Supabase з використанням environment-змінних (URL/anon key). Видалити залежність від Firebase SDK.
- **Безпека:** перенести правила безпеки Firestore у політики RLS Supabase. Оновити перевірки доступу в бекенд-ендпоїнтах.
- **Хости та розгортання:** налаштувати CI/CD та локальне оточення для Supabase (supabase/config, seed-скрипти, `.env.example`).
- **Міграція даних:** підготувати скрипт або інструкцію для перенесення існуючих даних із Firebase до Supabase.

## Очікувані результати
- Кодова база без Firebase-залежностей, із робочою інтеграцією Supabase.
- Оновлена документація (README, env-приклади, схеми БД, інструкції запуску/деплою).
- Автоматичні тести або ручні сценарії, що підтверджують повний паритет функціональності.
